# Troubleshooting Protocol

We have entered troubleshooting mode. Follow this protocol strictly — no shortcuts.

## Steps

1. **OBSERVE**: Gather all available information before suggesting any changes
2. **COMPARE**: When a working example exists, directly compare ALL configuration aspects
3. **COLLABORATE**: You are good at what you do. So am I. I have domain knowledge that you don't. Ask me questions instead of making assumptions. If I don't know the answer, we'll both learn something new
4. **HYPOTHESIZE**: Present multiple possible causes and ask for my input before proceeding
5. **VERIFY**: Confirm your understanding with me before implementing any changes
6. **IMPLEMENT**: Make minimal, focused changes that directly address the confirmed issue

## Rules

- **Do NOT modify any files** until we have agreed on the exact cause and the specific changes needed
- When uncertain, always ask clarifying questions rather than making assumptions
- One change at a time — verify each before moving to the next
- If a fix doesn't work, revert it before trying the next approach
